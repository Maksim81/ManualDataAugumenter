📄 Регламент разработки Dashboard: Training Centre
1. Стек технологий и окружение
Язык: Python 3.10+
Интерфейс: Streamlit (Layout: Wide)
База данных: MySQL (локальный сервер)
Драйвер БД: SQLAlchemy + PyMySQL
Схема: training_centre

2. Структура Базы Данных (Справочник)
Главная таблица: customer (PK: num)
Связанные таблицы (1:1 или 1:N): * customer_email (FK: customer_id)
customer_phone (FK: customer_id)

...остальные 8 таблиц добавляются по мере описания...

Системные таблицы: audit_trail (для логирования правок)

3. Правила обработки данных (Business Logic)
Принцип "Чистоты": Если поле в форме пустое, скрипт не должен затирать данные в базе пустой строкой. Обновлять только то, что введено.
Метод Upsert: Для связанных таблиц всегда использовать конструкцию:

SQL
INSERT INTO table (customer_id, field) VALUES (:id, :val)
ON DUPLICATE KEY UPDATE field = :val
Валидация: Все строковые значения обрабатывать методом .strip().

4. Правила Аудита
Любое изменение данных в таблицах customer, customer_email, customer_phone должно сопровождаться записью в audit_trail.

Запись создается только в том случае, если new_value != old_value.

Формат лога: id, customer_id, field_name, old_value, new_value, changed_at.

5. Требования к UI (Интерфейс)
Фильтрация: В основном списке показывать только тех клиентов, у которых есть хотя бы один NULL в ключевых полях (согласно вашему SQL-запросу).

Сортировка: Всегда ORDER BY c.num DESC (новые клиенты вверху).

Формы: Группировать поля ввода по смыслу (Личные данные / Контакты / Доп. инфо) в колонки st.columns.

Обратная связь: После сохранения использовать st.success() и st.rerun() для мгновенного обновления списка.

6. Стиль кода
Использовать sqlalchemy.text для всех SQL-запросов (защита от SQL-инъекций).

Комментарии в коде — на русском языке.

Обработка ошибок через блоки try...except с выводом сообщения в st.error.
